log4j2

maven-dependency

(Mar, 2021) 기준 log4j 는 2.14.1 버전이 최신입니다.

# 핵심 라이브러리
log4j-api:2.14.1
log4j-core:2.14.1
log4j-web:2.14.1
log4j-slf4j-impl:2.14.1

# 보조 라이브러리
log4jdbc-log4j2-jdbc4.1:1.16
slf4j-log4j12:1.7.32
log4j:1.2.17

log4j2.xml

설정 파일의 기본 구조입니다.

<Configuration>
  <Properties> ... </Properties>
  <Appenders> ... </Appenders>
  <Loggers> ... </Loggers>
</Configuration>

<Appenders> 설정

<Appenders> 는 로그의 출력을 설정합니다.

<File> 혹은 <RollingFile> 를 바로 정의 하는 것이 기본이고,
<Routing>을 사용하여 ThreadContext 의 값에 따라 <File> 혹은 <RollingFile> 을 routing 할 수 있습니다.

  <Appenders>
    <Console> ... </Console>
    <Routing>
      <Routes>
        <Route>
          <RollingFile> ... </RollingFile>
        </Route>
        <Route>
          <File> ... </File>
        </Route>
      </Routes>
    </Routing>
    <Async>
      <AppenderRef> ... </AppenderRef>
    </Async>
  </Appenders>

<Routing> 과 ThreadContext

<Routing> 설정이 동작할 수 있도록 ThreadContext 을 어떻게 사용하는지에 대한 예시 입니다.

참고로 ThreadContext는 log4j 의 Lookups 기능의 Context Map 을 사용하는 것이며, Lookups 의 세부 내용은 다시 다루겠습니다.

<Routing> 설정

<Routing>
  <Routes pattern="$${ctx:userId}">
    <Route>
      <RollingFile>
      </RollingFile>
    </Route>
    <Route>
      <File> ... </File>
    </Route>
  </Routes>
</Routing>

log-format

포맷 설명
%m 내용
%d 시각 %d{HH:mm:ss, SSS}, %d{yyyy MMM dd HH:mm:ss, SSS}
%t 쓰레드
%n 개행 문자
%c 카테고리(패키지명), %c{2}일 경우 a.b.c 패키지는 b.c 로 출력
%C 클래스, %c{2}일 경우 a.b.c.Clazz 는 c.Clazz 로 출력
%F 파일명
%l caller 정보
%L caller 의 line number
%M 메소드명
%r uptime (msec)
%x NDC(Nested Diagnostic Context)
%X MDC(Mapped Diagnostic Context)
