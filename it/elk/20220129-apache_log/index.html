apache 로그

로그 포맷
<VirtualHost *:443>
  ServerName nexus
  
  # combined 포맷은 httpd.conf 에 정의되어 있음
  LogFormat "%h %l %u %t \"%r\" %>s %b  \"%{Referer}i\" \"%{User-Agent}i\"" combined
  CustomLog "|/usr/sbin/rotatelogs /outlog/WEB/httpd/nexus_ssl-access.log.%Y-%m-%d 86400 +540%" combined
  ...
로그 설명

combined 라는 이름의 "%h %l %u %t \"%r\" %>s %b \"%{Referer}i\" \"%{User-Agent}i\"" 형식을 풀이하면 이렇다.

포맷 설명
%h 원격 호스트 이름
%l ident 인증으로 알아낸 os 의 사용자명 rfc14
%u http 인증으로 알아낸 사용자ID
%r 요청 uri (%m %U%q %H 와 같다.)
%>s http status
%b 클라이언트에 보낸 http body 의 byte 크기
%{Referer}i http header 명 Referer의 값
%{User-Agent}i http header 명 User-Agent의 값

로그 포맷 추가

combined 포맷에 http header txid 를 추가 후 curl 테스트

txid 가 포함된 로그를 kibana grok debugger 에서 %{COMBINEDAPACHELOG} > %{COMBINEDAPACHELOG} \"%{GREEDYDATA:txid}\" 로 테스트

$ vi /etc/httpd/conf.d/vhost-nexus.conf
<VirtualHost *:443>
  ServerName nexus
  
  # combined_nexus 포맷에 "txid" 추가
  # LogFormat "%h %l %u %t \"%r\" %>s %b  \"%{Referer}i\" \"%{User-Agent}i\"" combined
  LogFormat "%h %l %u %t \"%r\" %>s %b  \"%{Referer}i\" \"%{User-Agent}i\" \"%{txid}i\"" combined_nexus
  # access log 의 포맷을 combined_nexus 로 변경
  CustomLog "|/usr/sbin/rotatelogs /outlog/WEB/httpd/nexus_ssl-access.log.%Y-%m-%d 86400 +540%" combined_nexus
  ...

kibana grok debugger 에서 txid 를 parsing 하기 위해 %{GREEDYDATA:txid} 를 사용함

  • logstash 테스트 준비

kibana grok debugger 에서 테스트가 완료된 패턴을 logstash 에 추가

# grok > match 의 message 에 \"%{GREEDYDATA:txid}\" 추가
$ vi /etc/logstash/conf.d/exam_filebeat.conf
filter {
  mutate {
    remove_field => ["agent"]
  }
  grok {
    #match => { "message" => "%{COMBINEDAPACHELOG}"  }
    match => { "message" => "%{COMBINEDAPACHELOG} \"%{GREEDYDATA:txid}\"" }
$

$ systemctl stop logstash
$ systemctl stop filebeat
$ vi /etc/filebeat/filebeat.yml

$ systemctl start logstash
$ systemctl start filebeat
