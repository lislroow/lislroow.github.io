KInit

was booting 과정에서 웹 어플리케이션이 초기화되는 시점은

`javax.servlet.ServletContainerInitializer`를 구현한 클래스의 `onStartup(Set<Class<?>> webAppInitializerClasses, ServletContext servletContext)` 메소드 호출

입니다.

javax.servlet.ServletContainerInitializer를 구현한 클래스는 spring-web.jar 에 포함된 SpringServletContainerInitializer 클래스이며, onStartup 메소드에서는 WebApplicationInitializer 구현체를 탐지하고 onStartup(ServletContext servletContext) 메소드를 호출하는 것으로 spring 웹 어플리케이션 초기화가 시작됩니다.

웹 어플리케이션의 초기화를 위해 호출받기 위해 spring-webmvc.jar 의 AbstractAnnotationConfigDispatcherServletInitializer 클래스를 상속받은 클래스 KInit 을 생성 합니다.

참고

servlet 3.0 이후부터는 ServletContext 초기화(웹 어플리케이션의 초기화)를 web.xml 파일 아닌 WebApplicationInitializer를 구현하여 프로그래밍 방식으로 초기화할 수 있게 되었습니다. (servlet 3.0 은 2011에 release 되었음)

  • WebApplicationInitializer 는 spring-web.jar 에 포함된 인터페이스이며, 정확히는 javax.servlet-api.jar 의 ServletContainerInitializer 인터페이스를 구현한 클래스가 호출이 되고 WebApplicationInitializer 인터페이스를 구현한 클래스가 호출되는 형식입니다.

  • spring4+ 기반에서는 ServletContainerInitializer 를 구현체가 아닌 WebApplicationInitializer 를 구현체를 생성하면 됩니다.

  • WebApplicationInitializer 를 상속받는 것보다 spring-webmvc.jar 에서 제공하는 AbstractAnnotationConfigDispatcherServletInitializer 를 상속받는 것을 권합니다.

    WebApplicationInitializer (spring-web.jar)
 └ AbstractContextLoaderInitializer (spring-webmvc.jar)
   └ AbstractDispatcherServletInitializer
     └ AbstractAnnotationConfigDispatcherServletInitializer
       └ KInit

onStartup(ServletContext servletContext) 메소드 구현

  • javax.servlet.ServletContext 에 filter 와 listener 를 추가함
  • listener: request-listener
  • filter: spring-security-filterchain-bean, multipart-filter, characterset-filter

getRootConfigClasses(), getServletConfigClasses() 메소드 구현

  • root-context 초기화 클래스: KInitRoot
  • dispatcher-servlet 초기화 클래스: KInitDispatcher, KInitDispatcherSwagger
    (dispatcher-servlet 에 설정하는 swagger는 복잡도를 낮추기 위해 분리함)
  • dispacher-servlet 의 mapping 을 “/” 로 설정
  • dispacher-servlet 은 ServletContext 에 servlet 으로 추가됨

root-context 와 dispatcher-servlet 에 대한 초기화가 진행됨
