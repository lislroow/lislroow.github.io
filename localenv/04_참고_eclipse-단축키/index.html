04_참고_eclipse-단축키

1. 이클립스 기본 단축키

이클립스에 기본으로 등록된 단축키 목록입니다.

중요도에 따라 ★를 표시했습니다.

1.1 Ctrl + K (★★★)

  • 블록 선택된 단어가 다음에 일치하는 곳을 찾아갑니다.

  • Ctrl + Shift + K 일 경우 역방향으로 실행됩니다.

  • 대소문자 구분 옵션을 설정하려면 Ctrl + F 찾기에서 Case sentive 옵션을 선택해야합니다.
    Whole word 옵션으로 온전한 단어일 경우에만 검색되는 옵션도 있습니다.

    새 창 보기



1.2 Ctrl + Alt + DOWN (★★☆)

  • 블록 선택된 라인(혹은 커서가 있는 행)을 복제합니다.

  • DOWN 은 방향키를 의미합니다.

  • DOWN 일 경우 아래에 복제하고, UP 일 경우 위에 복제합니다.

    새 창 보기

1.3 Ctrl + Alt + A (★★★)

  • 블록 선택을 반전시키는 기능입니다.

  • 일반 모드일 경우에는 행을 선택하는 형태이고, 컬럼 단위로 선택을 하기 위해 사용합니다.

    새 창 보기

1.4 Ctrl + Shift + G (★★★)

  • 유틸리티 클래스를 리팩토링 할 경우 영향도 체크를 위해 반드시 익혀야 합니다.

  • References in Workspace 라는 기능입니다.
    기본 설명은 Search for references to the selected element in the workspace 라고 되어 있습니다.

  • java 의 클래스, 메소드, 변수가 참조되고 있는 파일의 라인을 찾아 줍니다.

  • 주로 프레임워크/공통 개발 시 제공된 클래스가 업무서비스의 어디서 참조되고 있는지를 검색해야 할 때 사용하면 좋습니다.

    새 창 보기



1.5 Alt + LEFT (★☆☆)

  • 이전에 활성화 했었던 파일을 re-open 해줍니다.
  • Ctrl + W 로 에디터에 열려진 파일을 닫은 직후 다시 열어야할 때 유용합니다.
  • RIGHT 일 경우는 LEFT 의 반대 방향입니다.


1.6 Ctrl + PAGEUP (★☆☆)

  • 현재 활성화된 탭의 왼쪽 탭을 활성화 시킵니다.
  • 에디터에 매우 많은 파일이 열려있을 경우 탭간의 이동을 마우스 클릭으로 이동하기 번거로울 경우 사용할 수 있습니다.
    개발의 집중도를 높이기 위해 관련없는 파일들은 수정이 끝나면 Ctrl + W 로 탭을 닫는 습관을 들이도록 해야겠습니다.
  • PAGEDOWN 일 경우는 PAGEUP 의 반대 방향, 오른쪽 탭을 활성화합니다.


1.7 Ctrl + Q (☆☆☆)

  • 이전에 수정했던 파일의 라인으로 이동합니다.

    새 창 보기

1.8 Ctrl + T (☆☆☆)

  • java 클래스의 상속 구조를 봐야하는 경우 사용됩니다.
    extends 에 대한 구조를 보여주며, implements 에 대한 구조는 보여주지 않습니다.

  • 프레임워크/공통 개발 시 오픈소스 라이브러리를 상속받고 구현체 클래스를 생성하는 경우가 많기에 알아둬야 겠습니다.

    새 창 보기

1.8 Ctrl + L (★☆☆)

  • editing 중인 파일의 해당 라인으로 이동하는 기능입니다.

    새 창 보기

1.9 Ctrl + Shift + R (★★★)

  • 이클립스 프로젝트에서 파일명으로 파일을 찾아서 open 하는 기능입니다.

  • 매우 사용 빈도가 높기에 필수로 기억해둬야 할 것입니다.

    새 창 보기

1.10 Ctrl + = (☆☆☆)

  • 이클립스 에디터의 font 크기를 변경하는 기능입니다. (= 은 더 크게, -는 더 작게)
  • 메뉴를 통해 font 크기를 변경하려면,
    Window > Preferences > General > Appearance > Colors and Fonts 로 이동한 다음
    Basic > Text Font 에서 변경할 수 있습니다.
  • 코드 리뷰 시간에 일시적으로 font 크기를 조절해야 할 경우 이 단축키를 사용할 것을 권합니다.
  • 단축키 Ctrl + = 에서 = 는 키보드의 숫자 라인에 + 와 같이 있는 키 입니다.


1.11 Ctrl + Alt + Shifh + L (★★☆)

  • 이클립스에 기본 탑재된 Quick Search 기능입니다.
    이클립스 InstaSearch 플러그인과 유사합니다.

  • 검색할 문자열을 블록 선택한 다음 Ctrl + Alt + Shifh + L 을 입력하면 팝업이 open 되면서 검색이 시작됩니다.

    새 창 보기



2. svn 단축키

2.1 Ctrl + Alt + L (★★★)

  • 현재 editing 중인 파일이 svn 저장소에 commit 된 파일과 비교를 합니다.

  • commit 을 하기전에는 무엇이 변경되었는지 빠르게 확인을 하기 위해 자주 사용되는 diff 단축키 입니다.

    새 창 보기



2.2 Ctrl + Alt + S (★★★)

  • svn 저장소와 동기화 단축키 입니다.

  • project explorer 에서 이클립스 프로젝트를 선택한 다음
    우클릭 > Team > Synchronized with Repository 를 선택하는 것과 동일합니다.
    마우스 클릭으로 sync 를 하기 위해서는 번거롭다고 생각하시면 익혀두셔야 겠습니다.

  • svn-commit(Ctrl + Alt + C), svn-update(Ctrl + Alt + U) 에 대한 단축키도 익혀두시면 좋겠습니다.

    새 창 보기




3. anyedit 플러그인 단축키

3.1 Ctrl + Alt + K (★☆☆)

  • camelcase 와 underscore-case 를 변환해주는 기능입니다.

  • 두 가지 case 에 대한 예시 입니다.

    # e.g 
employName 은 camelcase 이고, employ_name 는 underscore-case 입니다.

  • oracle 에서는 컬럼에 대해 대소문자가 구분이되지 않기 때문에 underscore-case 표기법을 사용하게 됩니다.
    java 에서는 oracle 컬럼에 대응되는 변수를 생성할 때
    camelcase 를 사용하는 경우가 일반적이므로 이 기능의 활용도가 있을 것으로 예상합니다.

  • 사용법은 변환할 문자열을 블록 선택한 다음 Ctrl + Alt + K 를 입력하면 됩니다.




4. 사용자 지정 단축키

개발을 하면서 자주 사용되는 기능에 대해 설정한 단축키 입니다.

사용자 지정 단축키는 User 부분에 U 라고 표시가 되어 있습니다.

만일 기존 단축키와 충돌이 발생할 경우 C 라는 문자가 같이 포함되어 있으며,

해제 하고자 하는 명령을 선택하고 Unbind Command 버튼을 클릭하면 해제할 수 있게 됩니다.

새 창 보기

4.1 Ctrl + Alt + H (★★☆)

  • project explorer 에서 선택된 파일 혹은 디렉토리의 svn 에 commit 된 history 를 볼 수 있습니다.

  • 특정 파일을 편집 상태에서도 Ctrl + Alt + H 를 입력하면, 편집 중인 파일의 svn-history 를 볼 수 있습니다.

    새 창 보기



4.2 Alt + C (★☆☆)

  • 이클립스 프로젝트를 다시 build(compile) 해야할 때 clean 을 사용되는 단축키 입니다.

  • 프로젝트를 선택한 상태에서 Alt + C 를 입력하면 선택된 프로젝트가 Clean 대상으로 체크가 되어있습니다.

  • java 파일을 editing 하는 상태에서 Alt + C 를 입력하면 현재 파일을 포함하는 프로젝트가 Clean 대상으로 체크됩니다.

    새 창 보기



4.3 Alt + U (★★★)

  • maven 프로젝트에서는 pom.xml 이 변경이 있거나, 의존성 library 의 최신 버전을 받아오기 위해 maven-update 를 자주 하게 됩니다.
  • 이클립스 기본 단축키는 Alt + F5 이며, F5 키가 접근이 용이하지 않아 Alt + U 단축키를 별도로 추가했습니다.

Preferences 창에서 General > Keys 를 선택하면 단축키 설정 화면이 보입니다.
