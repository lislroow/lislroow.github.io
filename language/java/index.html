Collection vs Stream

Collection Stream
Iterator로 모든 요소를 순회 해야한다. filter 로 일부 요소들만 순회할 수 있다.
메모리에 모든 요소가 load 된 상태에서 결과를 계산한다. 계산식을 미리 적어두고 계산시에 람다식으로 jvm에 넘긴다.
메모리 사용량이 늘어난다. 메모리 사용량이 줄어든다.

  • 함수형 인터페이스: 추상 메서드가 1개인 인터페이스. Predicate, Supplier, Consumer, Function, Comparator 가 있음
  • Lambda식은 함수형 인터페이스를 화살표 ->로 표현
  • method reference 또는 double colon operator(::) 를 사용할 수 있음
  • Stream은 데이터의 가공 및 처리를 chaining 형태의 api를 제공하는 인터페이스
  • Lambda식에서는 접근하는 외부 지역변수는 final 선언과 같이 변경되어서는 안 됨.
  • stream 객체는 재사용 불가, 최적화 안 됨.
initial operation
  • Arrays.stream(arr)
  • list.stream()
intermediate operation
  • filter()
  • map():
  • flatMap(): 중첩 구조를 한 단계 제거하고 map() 과 같은 처리를 함
  • sorted()
  • distinct()
terminal operation
  • count()
  • forEach()
  • collect()
  • anyMatch(), allMatch()
